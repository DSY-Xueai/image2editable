# 通用视觉元素拆分与 16:9 PPT 设计

## 目标

将任意扁平图片转换为可编辑 PPT 时，保持现有文字识别与文字重建逻辑不变，同时把所有非文字视觉元素拆成独立、透明、无文字的 PNG 图层，并生成独立的干净背景。输出 PPT 固定为 16:9，元素在原始位置重新组合时不得出现残缺、重影、明显边缘污染或比例失真。

测试图片只用于回归验证。实现不得包含针对特定人物、表格、卡片、国旗、颜色、坐标或版式的规则。

## 已确认的约束

- 每个有意义的视觉元素独立输出，不能把相互接触的不同对象合并。
- 每个视觉组件使用透明底，不包含整页矩形背景。
- 视觉组件不得保留任何原始文字像素。
- 文字继续由现有 OCR 和可编辑文本框负责，不修改文字识别、合并、字体或样式逻辑。
- 相互遮挡的对象只保留原图中可见部分，不生成或猜测被遮挡区域。
- PPT 保留一张独立、无文字、无前景元素的干净背景层。
- PPT 页面固定为 13.333 × 7.5 英寸，即标准 16:9。
- 输入图片不得被拉伸或裁切。
- 无法达到质量要求时明确失败并保留诊断结果，不能静默回退为整页压平图片。

## 视觉元素定义

视觉元素是能够作为一个整体理解和移动的可见对象或设计实体，包括人物、商品、图标、照片区域、插画、形状、色块、边框、装饰线和其他独立实例。

页面级底色、渐变、纹理或场景承载层属于背景。背景单独保留，不得混入前景组件。一个对象被其他对象遮挡后，其多个可见片段仍可属于同一个元素，但不补全不可见区域。

## 根因

当前前景拆分主要依赖颜色差分、边缘和连通域。连通域只能判断像素是否连接，无法判断两个相互接触的区域是否属于不同语义对象。密集页面还会触发文字优先降级，显式清空组件并把非文字内容保留在背景中，因此虽然文字可编辑，视觉元素没有独立图层。

当前 PPT 组装使用原图宽高比设置页面尺寸。测试图宽高比为 0.8003，生成页面为 13.333 × 16.660 英寸，不是 16:9。

## 方案选择

采用“通用实例分割为主、几何检测补漏、唯一像素归属和重组验证兜底”的混合方案。

不采用纯 OpenCV 增强方案，因为分水岭、超像素和轮廓层级仍无法可靠区分相互接触且颜色相近的语义对象。不采用仅依赖单次基础模型输出的方案，因为它容易漏掉细线和扁平色块，也会产生重复、父子和重叠蒙版。

## 核心模型

主分割器使用 SAM 2.1 Large 的自动蒙版生成能力。它无需预先知道对象类别，适合对整张图片生成通用候选蒙版。

模型通过官方 Python API 作为外部依赖调用，不复制或修改 SAM 2.1 源码。模型权重下载到用户缓存目录，不提交 Git。自动检测 CUDA；没有 CUDA 时允许 CPU 推理并提示速度较慢。

## 处理流程

### 1. 文字锁定

调用现有 `detect_text`，取得文字项和文字蒙版。现有 `scripts/text_detect.py` 保持冻结。视觉拆分只消费它的输出，不改变检测、合并、字体、字号、颜色、粗体或对齐逻辑。

### 2. 多尺度候选蒙版

SAM 2.1 分别处理整图和重叠局部裁剪，生成不同尺度的候选蒙版。局部裁剪用于发现整图推理容易忽略的小图标、细节和紧密相邻对象。

现有 OpenCV 能力改为补充检测器，只负责补充基础模型容易漏掉的细线、直线边框、封闭几何形状和稳定纯色区域，不再使用连通域结果决定最终对象身份。

### 3. 候选规范化

所有候选统一转换为原图坐标，删除面积过小的噪声和覆盖整页的背景候选。根据交并比、轮廓相似度和包含关系识别重复蒙版、父子蒙版和部分重叠蒙版。

重复蒙版只保留质量最高的一份。父子蒙版不会直接合并：子元素拥有自己的可见像素，父元素扣除被子元素占用的区域。相互接触但边界和视觉特征不同的候选保持为独立实例。

### 4. 唯一像素归属与层级

为全部候选建立包含和遮挡图，确定背景到前景的层级。除透明抗锯齿边缘外，每个可见像素只能归属于一个视觉元素。

底层对象在被上层对象遮挡的位置不保留重复像素。系统只保存底层对象当前可见的部分，不推测隐藏内容。这样组件在原位置组合时不会因为重复像素产生重影。

### 5. 文字清除

文字蒙版用于定位需要从视觉组件中移除的字形像素。组件 RGB 在文字位置执行局部修复，Alpha 仍由元素形状决定，避免直接挖空造成残缺。最终组件不得含有可识别的原始文字内容。

此步骤不修改可编辑文字框，只清理视觉图层中的栅格文字。

每个导出的组件会再次调用现有 OCR 做文字残留检查。发现文字时只重新清理该组件；限定次数后仍有文字则转换失败，不把含文字的组件写入 PPT。

### 6. 边缘精修

在每个最终蒙版边缘执行局部 Alpha Matting、颜色去污染、抗锯齿和小孔修复。边缘处理只使用原图已有像素和局部统计，不补全被遮挡对象。

输出组件为紧边界 RGBA PNG，并记录原始坐标、宽高、层级顺序、面积和质量分数。

### 7. 干净背景

使用所有最终视觉蒙版和文字蒙版从原图中移除前景与文字，再对缺失区域进行背景修复。背景必须位于最底层，且不得残留可辨识的前景对象或文字重影。

当前“密集页面直接保留全部非文字内容”的降级不再作为成功输出路径。

### 8. 重组验证

按记录的原始坐标和层级把干净背景、视觉组件和现有文字框重新组合。质量检查重点比较非文字区域：

- 检测漏掉的显著区域，并对局部区域重新生成候选蒙版。
- 检测重复覆盖，重新分配唯一像素归属。
- 检测明显边缘差异，重新执行边缘精修。
- 检测组件中的文字残留。
- 检测背景中的前景或文字重影。

限定次数的局部修正后仍不能通过检查时，转换明确失败并保留诊断目录，不生成看似成功但不可编辑的压平结果。

## 16:9 坐标与背景延展

PPT 固定为 13.333 × 7.5 英寸。每张输入图片使用同一个等比 `contain` 变换：

- 缩放比例取页面可用宽度比和高度比中的较小值。
- 内容区域在页面水平和垂直居中。
- 背景、视觉组件、文字框和参考页共享同一缩放比例与偏移量。
- 不拉伸、不裁切任何输入内容。

非 16:9 输入产生的空余区域由干净背景的边缘颜色或模糊延展填充。延展只发生在已移除文字和前景的背景上，不复制人物、图标或其他视觉元素。

多图输入时，所有幻灯片仍使用同一个 16:9 页面尺寸，每张图片独立计算 `contain` 变换。

## 失败处理

以下情况不得静默降级：

- SAM 2.1 未安装、模型权重不可用或模型初始化失败。
- 最终蒙版仍存在大面积重复归属。
- 视觉组件仍包含明显文字。
- 背景仍包含明显前景或文字重影。
- 重组结果无法达到质量阈值。

失败时保留候选蒙版、最终蒙版、像素归属图、背景、重组图、差异图和质量报告，并返回可执行的错误说明。

## 验证标准

自动测试覆盖：

- 相互接触的两个对象仍输出两个组件。
- 父元素与子元素不合并。
- 非边缘像素不会归属于多个组件。
- 每个组件为透明底 RGBA PNG。
- 组件中没有 OCR 文字像素。
- 背景中没有前景和文字重影。
- 透明边缘没有明显白边、黑边、锯齿或孔洞。
- 遮挡对象只保留原图可见部分。
- PPT 页面严格为 16:9。
- 任意输入比例均不拉伸、不裁切。
- 重组后的非文字区域满足视觉差异阈值。

测试使用合成接触对象、嵌套对象、带文字对象、细线和纯色几何图形，以及多种真实图片。真实测试图只作为回归输入，不进入任何算法分支。按照项目现有约定，`tests/` 和 `Course.md` 仍作为本地工程资料，不提交 GitHub。

## 非目标

- 不把栅格视觉元素自动重建为 PowerPoint 原生矢量图形。
- 不生成被遮挡对象的隐藏区域。
- 不修改现有文字识别和文字样式算法。
- 不承诺对未通过质量检查的图片生成降级压平版本。
