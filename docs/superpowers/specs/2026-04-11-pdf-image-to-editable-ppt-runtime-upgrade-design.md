# PDF/Image To Editable PPT Runtime Upgrade Design

## 目标

将现有的 `pdf-image-to-editable-ppt` 骨架升级为真实可运行的转换管线，使其具备以下能力：

1. 处理单张图片、单页 PDF、多页 PDF。
2. 为每一页生成视觉保真的 PPT 页面。
3. 在不破坏视觉完整性的前提下，尽可能提取文字与图片为可编辑对象。
4. 支持按页动态选择“原生文本提取”与 “OCR”。
5. 支持 PDF 原生坐标优先、像素坐标兜底的 PPT 对位。
6. 在后续阶段扩展复杂矢量与效果映射能力，但不牺牲保真基线。

## 当前基础

当前项目已具备：

- skill 目录结构
- `SKILL.md` 核心规则
- 页面与元素数据模型
- 保守筛选逻辑
- 最小 `.pptx` 输出能力
- 渲染/文字提取/图片提取脚本接口占位
- 对应测试骨架

本次升级的重点是把“接口占位”替换成“真实工作管线”。

## 总体路线

采用“两阶段设计，一阶段优先实现”的方案：

- 阶段一：稳定主链路
- 阶段二：高难增强层

阶段一先做到“稳定可用 + 不破坏视觉”；阶段二再提高复杂页面的可编辑覆盖率。

## 依赖策略

采用混合依赖策略：

- 优先 Python 纯库
- 必要时允许系统工具兜底

### 阶段一优先依赖

- `PyMuPDF`
  - PDF 渲染
  - 页面尺寸
  - 原生文本块提取
  - 原生图片对象提取
  - PDF 原生坐标获取
- `Pillow`
  - 图像读写
  - 图片裁切
  - 页面图尺寸处理
- `PaddleOCR`
  - 图片输入 OCR
  - 扫描页 OCR
  - 无可选文本页 OCR
- `python-pptx`
  - PPT 生成
  - 文本框插入
  - 图片对象插入
  - 幻灯片尺寸设置

### 可选系统工具兜底

- `Tesseract`
- `Poppler`
- 其它后续必要工具

但第一版不把系统工具设为强依赖。即使系统工具不可用，也必须能输出保真底图 PPT。

## 输入处理策略

### 图片输入

- 一张图片默认对应一张幻灯片。
- 图片本身作为页面底图。
- 页面文字通过 OCR 获取。
- 图片对象通过版面分析后高置信裁切获取。

### PDF 输入

- 一页 PDF 默认对应一张幻灯片。
- 多页 PDF 必须逐页处理。
- 每页先读取 PDF 原生宽高、坐标信息、原生文本与原生图片资源。
- 若用户明确要求切分长页，则允许切分为多张幻灯片，但每段仍保留对应底图。

## 文本提取策略

采用按页动态策略：

1. 对于 PDF 的每一页，先检测是否存在可选文本。
2. 若存在可选文本，则优先做原生文本提取。
3. 若不存在可选文本，或原生提取质量明显不足，则退回 OCR。
4. 对图片输入，直接走 OCR。

这意味着同一个多页 PDF 中：

- 某些页可走原生文本
- 某些页可走 OCR
- 不要求整个文档只能使用一种文本提取方式

## 图片提取策略

采用分级策略：

1. 先提取 PDF 原生嵌入图片对象。
2. 若版面分析能高置信识别独立图片区块，则允许从页面图中裁切提取。
3. 若图片与背景、蒙版、透明效果或其它元素耦合过强，则保留到底图层。

图片提取不得引入：

- 错误裁切
- 透明边缘损伤
- 层级错乱
- 比例失真

## 坐标映射策略

采用双基准坐标体系：

### PDF 原生对象

- 原生文字与原生嵌入图片优先保留 PDF 原生坐标。
- 再映射到 PPT 页面坐标。

### OCR / 裁切对象

- 基于渲染页图获得像素坐标。
- 再按页面尺寸比例映射到 PPT 坐标。

### 统一原则

- 每张 PPT 幻灯片尺寸与源页宽高比例一致。
- 所有对象最终都归一到同一 PPT 坐标空间。
- 不允许不同坐标体系混用导致对象漂移。

## 页面生成流程

阶段一每页执行如下流程：

1. 生成该页高保真底图。
2. 提取原生文本、OCR 文本候选。
3. 提取原生图片和高置信图片区块。
4. 标准化为统一的 `PagePlan` / `TextBlock` / `ImageBlock`。
5. 根据置信度和保真规则进行过滤。
6. 先写入 PPT 底图层。
7. 再叠加高置信文字层。
8. 再叠加高置信图片层。

## 保守提取规则

### 文字

文字仅在以下条件同时成立时才转为编辑层：

- 位置稳定
- 行块边界稳定
- 字号可可靠推导
- 颜色可提取
- 对齐方式明确
- 预计不会出现换行或字距明显漂移

只要有明显风险，就保留到底图层。

### 图片

图片仅在以下条件满足时提升为编辑层：

- 边界清晰
- 尺寸稳定
- 坐标可还原
- 不依赖复杂透明叠加

否则保留到底图层。

## 回退策略

### 块级回退

单个文字块或图片块不可靠时，仅回退该块。

### 页级回退

某页整体提取不可靠时，该页只输出底图层。

### 文档级保底

即使整份文档无法可靠提取任何可编辑对象，也必须输出逐页保真的 PPT。

## 阶段一实现范围

阶段一必须实际接入：

- 真实 PDF 渲染
- 多页 PDF 页面遍历
- 原生文本提取
- OCR 文本提取
- 原生图片提取
- 高置信图片区块裁切
- 幻灯片尺寸与页面比例映射
- 底图写入 PPT
- 文本框写入 PPT
- 图片对象写入 PPT
- 保守过滤和回退逻辑

阶段一不要求：

- 复杂矢量自动转形状
- 阴影、发光、透明混合模式重建
- 任意字体的完美替代
- 所有复杂页面都高比例变成原生可编辑对象

这些内容只要不可靠，就留在底图层。

## 阶段二增强范围

阶段二在不破坏保真基线的前提下增强：

- 复杂矢量路径转 PPT 形状
- 阴影、描边、透明度、发光等效果映射
- 字体拟合与替代策略
- 更细粒度版面分析
- 更复杂图片区块识别
- 对海报、杂志页、复杂版式的增强重建

阶段二原则：

- 只有映射可信才进入编辑层
- 不可信内容仍留到底图层
- 永远不允许为提高可编辑率破坏视觉完整性

## 模块边界

建议按以下责任划分：

- `render_pdf_pages.py`
  - PDF 渲染、页面尺寸、页面底图输出
- `extract_text_layout.py`
  - 原生文本检测与抽取
  - OCR 调用
  - 统一文字块标准化
- `extract_images.py`
  - 原生图片抽取
  - 高置信图片区块裁切
  - 图片块标准化
- `filtering.py`
  - 置信度筛选
  - 块级回退判断
- `build_ppt.py`
  - 幻灯片尺寸设定
  - 背景层写入
  - 文本框和图片对象写入
  - 坐标映射到 PPT

## 错误处理

- 缺少 OCR 依赖时，不中断整份转换，应退为保真底图输出。
- 单页提取失败时，不影响其它页面。
- 单个对象提取失败时，不影响整页底图与其它对象输出。
- 页面级异常必须带上页码上下文，便于后续定位。

## 测试重点

阶段一至少覆盖以下输入：

- 单页文本型 PDF
- 多页混合 PDF（部分页可选文本，部分页扫描）
- 单张海报图片
- 扫描 PDF
- 含嵌入图片的 PDF
- 需要长页切分的图片或 PDF

测试目标：

- 输出 PPT 可打开
- 幻灯片页数正确
- 每页尺寸比例正确
- 底图完整
- 高置信文字/图片对象可写入
- 低置信对象能正确回退

## 风险与约束

- OCR 对低清扫描件和复杂装饰字的稳定性有限。
- 字体缺失可能导致文本框难以做到完美拟合。
- `python-pptx` 对高级视觉效果支持有限。
- 复杂 PDF 的矢量与混合效果映射本身存在格式差异。

本设计接受这些限制，但不接受视觉完整性被破坏。
